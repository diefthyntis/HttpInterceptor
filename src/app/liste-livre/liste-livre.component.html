Apport théorique de l'exemple httpInterceptor

Cette exemple montre comment gérer une autorisation avec un token factice initiliasé à aLanNeSTnéle27aoAut19SoixanteTrEIze
La fonction d'autorisation est activé en ajoutant httpInterceptorProviders dans les providers de app.module.ts
Cela indique à ANGULAR qu'il doit gérer un service d'autorisation
Le résultat concrêt est l'ajout d'une ligne nommé "Authorization" dans l'entête de chaque requête Htttp, comme suit
Authorization: Bearer aLanNeSTnéle27aoAut19SoixanteTrEIze
Un intercepteur HTTP intercepte toutes les requêtes HTTP envoyées par votre application pour effectuer des tâches requises, comme l'ajout d'un header d'autorisation.
Un intercepteur Angular est une classe @Injectable qui implémente l'interface HttpInterceptor.
La méthode intercept() clone la requête reçue en ajoutant les modifications requises au clone.
intercept() passe ensuite la nouvelle requête à next.handle() pour lui permettre de continuer son chemin.
